Toybox¶

The Toybox class wraps up the configuration, simulator, and state into one convenient object that operates like an RL environment.

The code for this class can be found in /ctoybox/ctoybox/ffi.py.

class ctoybox.Toybox(game_name: str, grayscale: bool = True, frameskip: int = 0)¶

This is a stateful representation of Toybox – since it manages memory, we provide __enter__ and __exit__ usage for Python’s with-blocks:

Parameters

  • game_name – One of “breakout”, “space_invaders”, “amidar”, etc.

  • grayscale – Toybox can render directly to grayscale, saving time, default=True.

  • frameskip – When an action is submitted, for how many extra frames should it be applied? default=True

>>> with Toybox("amidar") as tb:
      print(tb.get_score())
>>> # the 'tb' variable only lives in the block.
apply_action(action_input_obj: ctoybox.input.Input)¶

Takes an Input action and applies it - unlike the ALE actions (which allow some permutations) this allows for fine-grained button pressing.

This applies the action k times, where k based on the frameskip passed to the Toybox constructor.

apply_ale_action(action_int: int)¶

Takes an integer corresponding to an action, as specified in ALE.

This applies the action k times, where k based on the frameskip passed to the Toybox constructor.
